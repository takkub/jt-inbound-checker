# J&T Inbound Checker

Chrome Extension ที่อ่าน QR code / barcode จากไฟล์ PDF ใบปะหน้าพัสดุ แล้วกรอกเลข waybill เข้าช่อง Shipping Inbound บนเว็บ J&T Express (`home.jtexpress.co.th`) โดยอัตโนมัติผ่าน DOM automation — ลดงาน manual ที่ต้องพิมพ์เลขพัสดุทีละชิ้น

## 1. J&T Inbound Checker คืออะไร

Extension นี้ทำงาน 2 ขั้นตอนต่อเนื่องกัน:

**ขั้นที่ 1 — อ่าน QR/barcode จาก PDF**
รับไฟล์ PDF ใบปะหน้าพัสดุ (1 ไฟล์ / หลายไฟล์ / หลายหน้า) แล้ว render แต่ละหน้าเป็น image ความละเอียดสูง จากนั้นถอดรหัส QR code หรือ barcode บนใบปะหน้าออกมาเป็นเลข waybill

**ขั้นที่ 2 — ยิงเข้าระบบ J&T**
นำเลข waybill ที่ได้ไปกรอกเข้าช่อง Waybill บนหน้า Shipping Inbound ของ J&T ทีละรายการ แล้วคลิกปุ่ม OK อัตโนมัติ รอรับผลจากระบบ J&T (ผ่าน / ไม่ผ่าน / ยิงซ้ำ / timeout) พร้อม retry อัตโนมัติสำหรับรายการที่ไม่ผ่าน

## 2. การติดตั้ง (เครื่องใหม่)

### 2.1 ดาวน์โหลด source code

> **สำคัญ:** ต้องใช้ **source code จาก repo** เท่านั้น ไม่ใช่ไฟล์จาก Releases tab  
> เพราะ Releases zip ไม่มีโฟลเดอร์ `tools/` ที่ใช้ติดตั้ง auto-updater

วิธีที่ 1 — ดาวน์โหลด ZIP:
1. ไปที่หน้า repo บน GitHub
2. คลิก **Code** → **Download ZIP**
3. แตก ZIP แล้วเก็บโฟลเดอร์ `extension/` ไว้ในที่ที่จะไม่ย้าย (เช่น `Documents/jt-checker/`)

วิธีที่ 2 — git clone:
```
git clone https://github.com/takkub/jt-inbound-checker.git
```

### 2.2 โหลด extension เข้า Chrome

1. เปิด Chrome แล้วไปที่ `chrome://extensions`
2. เปิดสวิตช์ **Developer mode** (มุมบนขวา)
3. คลิก **Load unpacked**
4. เลือก **โฟลเดอร์ `extension/`** ที่มีไฟล์ `manifest.json` อยู่ภายใน
5. Extension จะปรากฏใน toolbar พร้อมใช้งาน

### 2.3 ตรวจสอบ Extension ID

Extension ID ที่ถูกต้องคือ: **`oiglldeidblbehpagcjkjjojjpocgonb`**

ตรวจได้ที่ `chrome://extensions` → กล่อง J&T Inbound Checker → ดูบรรทัด ID  
หาก ID ไม่ตรง แสดงว่าโหลดผิดโฟลเดอร์

### 2.4 ติดตั้ง auto-updater (ออปชัน — Windows เท่านั้น)

ทำครั้งเดียวบนเครื่องใหม่เพื่อเปิดใช้ฟีเจอร์ "อัปเดตเดี๋ยวนี้" (คลิกเดียวอัปเดต+reload):

1. ดับเบิลคลิก **`tools/install-updater.bat`**
2. รอสักครู่จนหน้าต่างปิดเอง
3. ไปที่ `chrome://extensions` แล้วกด **Reload** (ไอคอนวนลูป) ที่ตัว extension 1 ครั้ง

> ถ้าย้ายโฟลเดอร์ `extension/` ไปที่อื่น ต้องรัน `install-updater.bat` ใหม่อีกครั้ง

## 3. วิธีใช้งาน

### 3.1 เปิดหน้า J&T ก่อน

**ต้องเปิด** `https://home.jtexpress.co.th` และไปที่หน้า **Shipping Inbound** (URL มี `#/mailPutIn`) ก่อนเสมอ จากนั้นจึงคลิก icon extension ใน toolbar

หากคลิก extension ขณะอยู่หน้าอื่น popup จะแสดงข้อความ:
> "กรุณาเปิดหน้า **J&T Shipping Inbound** แล้วกด extension icon อีกครั้ง"

หากมีการ scan รันอยู่ในแท็บ J&T อีกแท็บหนึ่ง จะแสดงแบนเนอร์สีฟ้าพร้อมปุ่ม **→ ไปที่แท็บ J&T** เพื่อสลับแท็บ

### 3.2 หน้าตา popup

Popup มี **2 แท็บ** ที่แถบสีน้ำเงินด้านบน:
- **Scan** — แท็บหลัก ใช้ยิง waybill
- **History (ไม่ผ่าน)** — ดูประวัติรายการที่ยิงไม่ผ่าน

มุมขวาบนของแถบแสดง badge เวอร์ชัน เช่น **`v4.4.2`**

### 3.3 วิธีป้อนข้อมูล — 2 แบบ

#### แบบที่ 1: อัปโหลด PDF ใบปะหน้า (แนะนำ)

1. กดปุ่ม **⬆ อัปโหลด PDF ใบปะหน้า** (สีเขียว)
2. เลือกไฟล์ PDF ได้หลายไฟล์พร้อมกัน
3. Extension จะ render ทุกหน้าแล้วถอด QR/barcode — ระหว่างนี้จะแสดงความคืบหน้าใต้ปุ่ม เช่น:
   - `อ่านหน้า 3/10 · เจอ 2 เลข`
4. เมื่อเสร็จจะแสดงสรุป เช่น:
   - `ได้ 8 เลขใหม่ | invoice.pdf: 8 เลข / 10 หน้า`
   - หากมีหน้าอ่านไม่ออก: `⚠ 2 หน้าต้องตรวจเอง: หน้า 3, 7` (ข้อความจะเป็นสีแดง)
5. เลขที่อ่านได้จะถูกใส่เข้า textarea ด้านล่างอัตโนมัติ (dedup กับที่มีอยู่แล้ว)

#### แบบที่ 2: วางเลข waybill ตรง ๆ

คลิกในช่อง textarea แล้ววางหรือพิมพ์เลข waybill — รองรับทั้งแบบบรรทัดละเลข และคั่นด้วยคอมมา:
```
EX0123456789TH
EX0123456790TH, EX0123456791TH
```
> Duplicate ใน textarea จะถูกกรองออกอัตโนมัติเมื่อกด Start

### 3.4 ฟิลด์ตั้งค่า

| ฟิลด์ | ค่า default | อธิบาย |
|---|---|---|
| **หน่วงเวลา (ms)** | `1000` | รอนานแค่ไหน (มิลลิวินาที) หลังจากยิง waybill แต่ละรายการก่อนยิงรายการถัดไป ลดได้ถ้าเน็ตเร็ว/J&T ตอบเร็ว แต่ต่ำเกินอาจทำให้ระบบ J&T ไม่ทัน |
| **Retry รอบ** | `2` | หลังยิงครบ queue รอบแรก ระบบจะ retry รายการที่ไม่ผ่านโดยอัตโนมัติกี่รอบ (0 = ไม่ retry) |

ค่าตั้งค่าจะถูกบันทึกอัตโนมัติทุกครั้งที่เปลี่ยน

### 3.5 ปุ่ม Start / Stop / Retry failed

| ปุ่ม | สี | ทำอะไร |
|---|---|---|
| **▶ Start** | น้ำเงิน | เริ่มยิง waybill ทุกรายการใน textarea ตามลำดับ จากนั้น retry อัตโนมัติตามจำนวนรอบที่ตั้ง |
| **■ Stop** | แดง | หยุดทันที (รายการที่กำลังยิงอยู่จะรอผลก่อน แล้วหยุดที่รายการถัดไป) |
| **↺ Retry failed** | ส้ม | ยิงซ้ำเฉพาะรายการที่ยังไม่ผ่าน (ใช้ได้หลัง Start เสร็จ และมีรายการ fail อยู่) |

### 3.6 Progress bar และข้อความความคืบหน้า

ใต้ปุ่ม controls มีแถบ progress สีน้ำเงิน (얇 4px) และข้อความทางขวา เช่น:
- `5/10 (50%)` — ระหว่างยิง
- `↺ Retry 1/2 — 3 รายการ` — ระหว่าง auto-retry
- `เสร็จ 10 รายการ ✓` — ทุกรายการผ่าน
- `เสร็จ 10 รายการ (ไม่ผ่าน 2)` — มีบางรายการไม่ผ่าน
- `หยุดแล้ว` — กด Stop

### 3.7 ตาราง Waybill / สถานะ / ข้อความ

| สถานะ | ไอคอน/สี | ความหมาย |
|---|---|---|
| **✓ ผ่าน** | เขียว | ยิงเข้าระบบ J&T สำเร็จ หรือเป็นรายการซ้ำที่เคยยิงแล้ว |
| **✗ ไม่ผ่าน** | แดง | J&T ตอบ error — ดูคอลัมน์ "ข้อความ" สำหรับรายละเอียด |
| **⟳ กำลังเช็ค** | น้ำเงิน | กำลังยิงอยู่ในขณะนี้ |
| **⚠ Timeout** | ส้ม | รอ 3 วินาทีแล้วระบบ J&T ไม่ตอบ (อาจ retry ได้) |
| **— รอ** | เทา | รอคิวอยู่ ยังไม่ได้ยิง |

รายการที่ "ยิงซ้ำ" (waybill เคยถูกยิงเข้าระบบแล้ว) ระบบ J&T จะแจ้งผ่าน dialog หรือ toast — extension จะแปลงเป็น **✓ ผ่าน** พร้อมข้อความ "ยิงซ้ำ (เลขนี้เคยยิงเข้าระบบแล้ว)" เพื่อให้ batch ดำเนินต่อได้โดยไม่ติด

### 3.8 badge เวอร์ชันมุมขวาล่างบนหน้า J&T

เมื่อเปิดหน้า `home.jtexpress.co.th` จะเห็น badge กึ่งโปร่งใส มุมขวาล่างของหน้าเว็บ แสดง `J&T Checker v4.4.2` — ยืนยันว่า extension โหลดทำงานบนหน้านั้นแล้ว (ไม่ block การคลิก)

badge นี้จะแสดงความคืบหน้าแบบ `3/10` ใน Chrome tab badge icon ด้วย ระหว่างที่ scan รัน

### 3.9 แท็บ History (ไม่ผ่าน)

เก็บประวัติรายการที่ **ยิงไม่ผ่าน** (ไม่รวมที่ผ่านแล้ว) ข้ามรอบการใช้งาน

| องค์ประกอบ | หน้าที่ |
|---|---|
| ช่องค้นหา | กรองตามเลข waybill แบบ real-time |
| **⬇ Export CSV** | ดาวน์โหลดข้อมูล failure ทั้งหมดเป็นไฟล์ CSV |
| **🗑 Clear All** | ลบประวัติทั้งหมด (มี dialog ยืนยันก่อน) |
| ตาราง | แสดง Waybill / ข้อความ Error / เวลา (ใหม่สุดขึ้นก่อน) |

## 4. การอัปเดต

### 4.1 แบนเนอร์แจ้งเตือน

ทุกครั้งที่เปิด popup extension จะเช็คเวอร์ชันล่าสุดจาก GitHub อัตโนมัติ  
ถ้ามีเวอร์ชันใหม่ จะแสดงแบนเนอร์สีเหลืองด้านบน:

> 🔔 มี v4.5.0 ใหม่ — **กดโหลด** | **อัปเดตเดี๋ยวนี้** | · หลังโหลด: Extensions → J&T Inbound Checker → Reload

> **หมายเหตุ:** แบนเนอร์ใช้ข้อมูลจาก GitHub — หากเพิ่งปล่อยเวอร์ชันใหม่ อาจต้องรอ ~5 นาที ให้ cache หมดอายุก่อน

### 4.2 วิธีอัปเดต — 2 แบบ

**แบบที่ 1: อัตโนมัติ (Windows + ติดตั้ง updater แล้ว)**
1. กดปุ่ม **อัปเดตเดี๋ยวนี้** ในแบนเนอร์
2. รอสักครู่ — extension จะดาวน์โหลดไฟล์ใหม่, แตก zip, และ reload ตัวเองอัตโนมัติ
3. popup จะปิดแล้วเปิดใหม่ด้วยเวอร์ชันล่าสุด

**แบบที่ 2: manual (ทุก OS)**
1. กดลิงก์ **กดโหลด** ในแบนเนอร์ → ดาวน์โหลด zip
2. แตก zip แล้วคัดลอกไฟล์ทับลงในโฟลเดอร์ `extension/` เดิม
3. ไปที่ `chrome://extensions` → กด **Reload** (ไอคอนวนลูป) ที่ J&T Inbound Checker

| อาการ | สาเหตุ / วิธีแก้ |
|---|---|
| กด "อัปเดตเดี๋ยวนี้" แล้วขึ้น "ยังไม่ได้ติดตั้ง updater" | รัน `tools/install-updater.bat` แล้ว reload extension ที่ `chrome://extensions` — จากนั้นลองกดใหม่ |
| แบนเนอร์ไม่เด้งทั้งที่มีเวอร์ชันใหม่แล้ว | GitHub cache ยังค้างอยู่ รอประมาณ 5 นาทีแล้วปิด/เปิด popup ใหม่ |
| icon extension เป็นสีเทา / คลิกแล้วไม่ขึ้นอะไร | ไปที่ `chrome://extensions` กด **Reload** ที่ J&T Inbound Checker |
| ย้ายโฟลเดอร์ extension แล้ว "อัปเดตเดี๋ยวนี้" ไม่ทำงาน | updater บันทึก path เก่าไว้ ต้องรัน `tools/install-updater.bat` ใหม่อีกครั้ง |

| PDF อัปโหลดแล้วได้ "⚠ X หน้าต้องตรวจเอง" | หน้าเหล่านั้น QR/barcode เบลอหรืออ่านไม่ออก ต้องพิมพ์เลข waybill เองลงใน textarea |
| badge เวอร์ชันบนหน้า J&T ไม่แสดง | refresh หน้า J&T — content script ต้องโหลดหลัง extension พร้อม |

## 6. ข้อจำกัด

- **Auto-updater ใช้ได้เฉพาะ Windows + Chrome** (Native Messaging ต้องการ .bat/registry บน Windows)  
  ผู้ใช้ macOS / Linux ต้องอัปเดตแบบ manual เสมอ
- **ตัว extension** (การอ่าน PDF และยิง waybill) ใช้ได้กับ **ทุก OS ที่รัน Chrome**
- Extension ทำงานกับหน้า `home.jtexpress.co.th` เท่านั้น — ไม่รองรับหน้าอื่น
- การยิง waybill อาศัย DOM automation — หาก J&T เปลี่ยน UI อาจหยุดทำงาน รอ update จาก repo
- เวลา timeout ต่อ 1 รายการคือ 3 วินาที — เครือข่ายช้าหรือ J&T ตอบช้าอาจได้ Timeout แล้วค่อย retry
